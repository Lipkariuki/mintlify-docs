This page explains how to request and prepare for access to the **Smithsons Logistics API** without exposing private authentication details.

## What You Will Do

1. Confirm your use case and environment
2. Request private API access from Smithsons
3. Prepare your integration before onboarding credentials are issued

## Prerequisites

Before you begin, make sure you have:

- An active Smithsons account
- API access enabled
- A technical contact for onboarding
- A tool for testing HTTP requests (e.g., curl, Postman, or your backend application)

## Base URL

### Production
`https://api.smithsons.co.ke`

### Local development
`http://127.0.0.1:8000`

<Warning>
Public documentation does not include authentication payloads, credentials, or token usage details.
</Warning>

## Step 1: Request Private API Access

Contact the Smithsons team to request onboarding and private integration access.

Provide:

- Your company name
- Integration use case (orders, trips, finance visibility, etc.)
- Environment needed (sandbox/staging/production)
- Technical contact email
- Expected traffic/usage pattern (if known)

<Tip>
Private onboarding details are shared only with approved partners and internal teams.
</Tip>

## Step 2: Prepare Your Integration

Before credentials are issued, prepare:

- Your backend service for secure server-to-server calls
- Error handling and retry logic
- Request logging and monitoring
- A staging environment for integration testing

## Step 3: Validate Connectivity (Optional)

You can verify your application can reach the API host before onboarding is completed.

```bash
curl -I "https://api.smithsons.co.ke"
```
