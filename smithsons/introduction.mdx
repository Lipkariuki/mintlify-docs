## What Smithsons Helps You Do

- Manage customer transport orders and dispatch workflows
- Track trips and delivery progress in a structured way
- Record operational expenses per trip
- Manage users by role (Admin, Driver, Partner)
- Improve financial visibility and reporting across logistics operations

## Authentication at a Glance

All protected endpoints require a bearer token in the `Authorization` header.

```http
Authorization: Bearer YOUR_ACCESS_TOKEN
```

Tokens are issued through the login flow. See `/smithsons/authentication` for the full workflow and examples.

## Core Resources

| Resource | What it represents |
| --- | --- |
| Orders | Customer transport orders and fulfillment status |
| Trips | Executable trip records linked to orders |
| Expenses | Fuel, tolls, and other trip-related operational costs |
| Users | Admins, drivers, and partners using the platform |

## Example Request

Fetch all orders as an admin:

```bash
curl -X GET "https://api.smithsons.co.ke/admin/orders" \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN"
```

```json
[
  {
    "id": 12,
    "order_number": "DHL-2025-001",
    "destination": "Nairobi",
    "total_amount": 85000,
    "status": "completed"
  }
]
```
