Smithsons supports role-based access control across operational and admin workflows.

## Roles

| Role | Description |
| --- | --- |
| Admin | Manages operations, orders, trips, expenses, and users |
| Driver | Handles assigned trips and operational updates |
| Partner | Participates in partner-linked logistics and financial workflows |

## Common User Management Tasks

- Create staff or partner accounts
- Assign roles and permissions
- Update profile/contact details
- Enable or disable access
- Audit usage by role

## Security Notes

<Warning>
Restrict admin account creation to trusted operational staff only. Admin roles can affect dispatch, finance, and user access.
</Warning>

- Use strong passwords and secure reset flows
- Rotate credentials when staff changes occur
- Review role assignments periodically

## Example Request

```bash
curl -X GET "https://api.smithsons.co.ke/admin/users"
```
