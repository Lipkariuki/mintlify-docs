Smithsons Logistics uses JWT bearer authentication for protected endpoints.

## Authentication Flow

1. Send credentials to the login endpoint
2. Receive an `access_token`
3. Attach the token to the `Authorization` header
4. Call protected endpoints based on the user's role

## Login Endpoint

Use the Smithsons auth login route to obtain a token.

```bash
curl -X POST "https://api.smithsons.co.ke/auth/login" \
  -H "Content-Type: application/json" \
  -d '{
    "phone": "0722760992",
    "password": "YOUR_PASSWORD"
  }'
```

```json
{
  "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "token_type": "bearer",
  "user": {
    "id": 7,
    "name": "Operations Admin",
    "role": "admin"
  }
}
```

## Send the Token in Requests

Add the JWT to the request header:

```http
Authorization: Bearer YOUR_ACCESS_TOKEN
```

Example:

```bash
curl -X GET "https://api.smithsons.co.ke/admin/orders" \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN"
```

## Roles and Access

Smithsons enforces role-based access control.

| Role | Typical access |
| --- | --- |
| Admin | Full operational and management routes |
| Driver | Assigned trips and delivery-related actions |
| Partner | Partner-specific operational and financial visibility |

<Note>
Being authenticated does not guarantee access to every route. A valid token can still receive `403 Forbidden` if the user role is not allowed.
</Note>

## Best Practices

- Store tokens securely on the server side when possible
- Avoid exposing long-lived credentials in frontend code
- Log out users and clear tokens when sessions end
- Re-authenticate when the API returns token-expiry errors

## Common Authentication Errors

<AccordionGroup>
  <Accordion icon="triangle-exclamation" title="401 Unauthorized">
    The token is missing, malformed, expired, or signed with the wrong secret. Re-authenticate and retry the request.
  </Accordion>
  <Accordion icon="ban" title="403 Forbidden">
    The user is authenticated but lacks permission for the endpoint.
  </Accordion>
  <Accordion icon="circle-exclamation" title="422 Validation Error">
    The login request payload may be missing a required field such as `phone` or `password`.
  </Accordion>
</AccordionGroup>
